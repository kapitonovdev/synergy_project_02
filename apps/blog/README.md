# Practice Blog

Учебное приложение блога для кейс-задачи № 3. Данные хранятся в SQLite через Prisma Client, вход реализован через простую cookie-сессию.

## Функции

- регистрация, вход и выход пользователя;
- хранение пользователей, постов, тегов, подписок, комментариев и заявок доступа в SQLite;
- создание, редактирование и удаление собственных постов;
- публичные посты и посты `REQUEST_ONLY`;
- запрос доступа к закрытому посту, одобрение и отклонение автором;
- подписка на пользователей и лента подписок;
- фильтрация по тегам;
- комментарии к доступным постам.

## Запуск

```bash
pnpm install
pnpm --dir apps/blog prisma:generate
pnpm --dir apps/blog db:push
pnpm --dir apps/blog db:seed
pnpm --dir apps/blog dev
```

Приложение запускается на `http://127.0.0.1:3001`.

`.env`:

```env
DATABASE_URL="file:./dev.db"
```

## Сценарии проверки

1. Войти под `pavel@example.com`.
2. Создать пост с тегами и видимостью `PUBLIC`.
3. Создать пост с видимостью `REQUEST_ONLY`.
4. Выйти, войти под `anna@example.com` и запросить доступ к закрытому посту.
5. Войти под автором и одобрить заявку.
6. Подписаться на другого пользователя и проверить ленту подписок.
7. Добавить комментарий к доступному посту.

## Анализ выполнения

1. Пользовательский сценарий построен вокруг публикации и чтения постов.
2. Регистрация и вход сохраняют пользователей в SQLite, пароль хранится в виде SHA-256 хеша.
3. Модель видимости разделяет публичные записи и записи с доступом по заявке.
4. Заявки доступа реализуют request-only поведение без ручного переключения демо-профилей.
5. Подписки формируют персональную ленту на основе реальных записей БД.
6. Теги решают задачу сортировки и тематической навигации.
7. Комментарии расширяют взаимодействие между пользователями.

## Рекомендации по устранению ошибок

1. Для production заменить учебную cookie-сессию на полноценный auth-механизм.
2. Усилить хеширование паролей через соль и специализированный алгоритм.
3. Добавить серверную валидацию длины заголовков, текста и тегов.
4. Добавить пагинацию для длинной ленты.
5. Добавить журнал действий по заявкам доступа.
6. Добавить модерацию комментариев.
7. Покрыть основные сценарии e2e-тестами.
