# Practice Blog

Учебное приложение блога для кейс-задачи № 3.

## Функции

- регистрация/вход смоделированы переключателем демо-пользователей;
- создание, редактирование и удаление постов;
- публичные посты и посты «только по запросу»;
- подписка на пользователей;
- персональная лента на основе подписок;
- сортировка и фильтрация по тегам;
- комментарии к постам.

## Запуск

```bash
pnpm install
pnpm --dir apps/blog dev
```

Для подключения постоянного хранения:

```bash
cd apps/blog
cp .env.example .env
pnpm prisma:migrate
```

`.env`:

```env
DATABASE_URL="file:./dev.db"
```

## Анализ выполнения

1. Пользовательский сценарий построен вокруг публикации и чтения постов.
2. Добавлена модель видимости: публичные записи и записи с ограниченным доступом.
3. Подписки позволяют формировать персональную ленту без сложной рекомендательной системы.
4. Теги решают задачу сортировки и тематической навигации.
5. Комментарии расширяют взаимодействие между пользователями.
6. Prisma-схема отделяет доменную модель от интерфейса и готовит проект к БД.
7. Интерфейс адаптирован под рабочий сценарий: авторы слева, публикации в центре, лента справа.

## Рекомендации по устранению ошибок

1. Добавить серверную авторизацию и хранение паролей через безопасное хеширование.
2. Подключить Prisma Client к route handlers или server actions.
3. Добавить валидацию длины заголовков, текста и тегов.
4. Добавить пагинацию для длинной ленты.
5. Реализовать реальные запросы доступа к скрытым постам.
6. Добавить модерацию комментариев.
7. Покрыть основные сценарии e2e-тестами.
