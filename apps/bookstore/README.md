# Bookstore Practice

Web-версия книжного магазина для кейс-задачи № 4.

## Функции

- два интерфейса: пользователь и администратор;
- база любимых книг;
- просмотр каталога;
- сортировка по категории, автору и году;
- покупка книги;
- аренда на 2 недели, 1 месяц или 3 месяца;
- изменение цены, статуса и доступности книги;
- напоминания об окончании срока аренды.

## Запуск

```bash
pnpm install
pnpm --dir apps/bookstore dev
```

Для подключения постоянного хранения:

```bash
cd apps/bookstore
cp .env.example .env
pnpm prisma:migrate
```

`.env`:

```env
DATABASE_URL="file:./dev.db"
```

## Тестовые роли

- Пользователь - просмотр, покупка и аренда.
- Администратор - управление каталогом и напоминаниями.

## Анализ выполнения

1. Каталог построен как основной пользовательский сценарий магазина.
2. Роли разделяют обычные операции и административные функции.
3. Фильтры и сортировки позволяют быстро найти книгу.
4. Покупка и аренда отражают разные варианты получения книги.
5. Аренда хранит дату окончания и формирует напоминания.
6. Администратор может менять цену и доступность без редактирования кода.
7. Prisma-схема описывает пользователей, книги, покупки и аренды.

## Рекомендации по устранению ошибок

1. Добавить серверную проверку роли администратора.
2. Подключить транзакции для покупки и аренды.
3. Добавить автоматическую отправку email-напоминаний.
4. Ввести остатки экземпляров книг.
5. Добавить историю изменений цены.
6. Добавить поиск по названию.
7. Покрыть критические сценарии тестами.
