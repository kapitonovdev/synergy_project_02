# Bookstore Practice

Web-версия книжного магазина для кейс-задачи № 4. Каталог, роли, покупки, аренды и напоминания хранятся в SQLite через Prisma Client.

## Функции

- регистрация, вход и выход пользователя;
- роли `USER` и `ADMIN`;
- просмотр каталога книг;
- сортировка по категории, автору и году;
- покупка книги с сохранением записи в БД;
- аренда на 2 недели, 1 месяц или 3 месяца;
- расчет даты окончания аренды;
- административное изменение цены, статуса и доступности книги;
- список просроченных и приближающихся аренд;
- отметка напоминания как отправленного через поле `reminded`.

## Запуск

```bash
pnpm install
pnpm --dir apps/bookstore prisma:generate
pnpm --dir apps/bookstore db:push
pnpm --dir apps/bookstore db:seed
pnpm --dir apps/bookstore dev
```

Приложение запускается на `http://127.0.0.1:3002`.

`.env`:

```env
DATABASE_URL="file:./dev.db"
```

## Тестовые учетные записи

- `reader@example.com` / `123456` - пользователь.
- `admin@example.com` / `123456` - администратор.

## Сценарии проверки

1. Войти под `reader@example.com`.
2. Проверить сортировку каталога по категории, автору и году.
3. Купить книгу и убедиться, что она появилась в разделе покупок.
4. Оформить аренду на 2 недели, 1 месяц и 3 месяца.
5. Войти под `admin@example.com`.
6. Изменить цену, статус и доступность книги.
7. В разделе напоминаний отметить аренду как отправленную.

## Анализ выполнения

1. Каталог построен как основной пользовательский сценарий магазина.
2. Роли разделяют обычные операции и административные функции.
3. Пользователи и администраторы являются реальными записями SQLite.
4. Фильтры и сортировки позволяют быстро найти книгу.
5. Покупка и аренда создают отдельные записи в БД.
6. Аренда хранит дату окончания и формирует напоминания.
7. Администратор может менять цену, статус и доступность без редактирования кода.

## Рекомендации по устранению ошибок

1. Для production заменить учебную cookie-сессию на полноценную авторизацию.
2. Подключить транзакции для покупки и аренды.
3. Добавить автоматическую отправку email-напоминаний.
4. Ввести остатки экземпляров книг.
5. Добавить историю изменений цены.
6. Добавить поиск по названию.
7. Покрыть критические сценарии тестами.
