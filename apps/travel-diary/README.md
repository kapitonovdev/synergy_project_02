# Travel Diary Practice

Приложение «Дневник путешествий» для кейс-задачи № 5. Пользователи, путешествия и места сохраняются в SQLite через Prisma Client.

## Функции

- регистрация, вход и выход пользователя;
- создание собственной записи о путешествии;
- хранение местоположения, стоимости, оценки безопасности и URL изображения;
- отображение реального изображения по `imageUrl`;
- аккуратный fallback-блок, если URL изображения не указан;
- список мест для посещения;
- просмотр общей ленты записей других пользователей;
- фильтр «Мои записи» для текущего пользователя.

## Запуск

```bash
pnpm install
pnpm --dir apps/travel-diary prisma:generate
pnpm --dir apps/travel-diary db:push
pnpm --dir apps/travel-diary db:seed
pnpm --dir apps/travel-diary dev
```

Приложение запускается на `http://127.0.0.1:3003`.

`.env`:

```env
DATABASE_URL="file:./dev.db"
```

## Сценарии проверки

1. Войти под `pavel@example.com`.
2. Создать путешествие с местоположением, стоимостью, оценкой безопасности и `imageUrl`.
3. Добавить несколько мест для посещения через строку со списком.
4. Проверить отображение изображения в карточке.
5. Переключиться на «Мои записи».
6. Переключиться на общую ленту и увидеть записи других пользователей.
7. Создать запись без `imageUrl` и проверить fallback-блок.

## Анализ выполнения

1. Основной сценарий - создание и чтение записей о путешествиях.
2. Регистрация и вход заменяют старую демонстрационную смену профиля.
3. Пользователь может отделить свои записи от общей ленты.
4. Местоположение помогает структурировать путешествия географически.
5. Реальный `imageUrl` добавляет визуальную идентификацию маршрута.
6. Стоимость позволяет сравнивать поездки по бюджету.
7. Список мест для посещения превращает запись в практический маршрут.

## Рекомендации по устранению ошибок

1. Добавить загрузку файлов изображений вместо ручного URL.
2. Подключить геокодирование и карту.
3. Для production заменить учебную cookie-сессию на полноценную авторизацию.
4. Добавить приватные и публичные записи.
5. Реализовать редактирование и удаление путешествий.
6. Добавить фильтр по городу и стоимости.
7. Покрыть форму создания записи тестами.
